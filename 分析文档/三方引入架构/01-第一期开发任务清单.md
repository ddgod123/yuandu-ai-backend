# 第一期开发任务清单

> 目标：在 2~4 周内，把第三方工具接入做成“可灰度、可回退、可量化”的第一期版本。

---

## 1. 第一期边界

第一期只做下面 4 件事：

1. **接入 Gifsicle** 到 GIF 渲染后优化
2. **接入 PySceneDetect** 到候选召回前，先旁路后切流
3. **建立 VMAF 离线回归链路**，不进用户同步主链路
4. **补齐平台可观测性最小集**：Asynqmon + OpenTelemetry

**第一期不做：**

- 不做 Stable Diffusion / ComfyUI / 生成式增强主链路
- 不做 Real-ESRGAN 默认增强
- 不做 imgproxy 全量替换分发层
- 不重写 Processor 主架构

---

## 2. 研发拆分建议

## 2.1 Epic A：Runtime 能力探测扩展

### 目标

把当前 `ffmpeg / ffprobe` 运行时探测，扩成通用第三方工具探测器。

### 涉及文件

- `internal/videojobs/capabilities.go`
- `internal/handlers/video_jobs.go`
- `internal/config/config.go`

### 交付项

- 返回：`gifsicle_available`
- 返回：`pyscenedetect_available`
- 返回：`vmaf_available`
- 返回：`external_tools[]`
- 前台/后台可以直接看到工具可用性

### 完成标准

- `GET /api/video-jobs/capabilities` 能返回完整工具状态
- worker 启动日志能打印关键工具状态
- 工具不可用时，能明确 fallback 而不是静默失败

---

## 2.2 Epic B：Gifsicle 接入

### 目标

在 GIF 渲染后，增加 GIF 优化步骤。

### 涉及文件

- `internal/videojobs/processor_gif_render.go`
- `internal/videojobs/processor_animated_task.go`
- `internal/videojobs/gif_rerender.go`
- `internal/videojobs/gif_evaluation.go`
- `internal/handlers/video_quality_settings.go`
- `internal/videojobs/quality_settings.go`

### 子任务

#### B1. 新增工具封装

新增：

- `internal/videojobs/tool_runtime.go`
- `internal/videojobs/tool_gifsicle.go`
- `internal/videojobs/tool_gifsicle_test.go`

#### B2. 新增后台开关

在 quality settings 增加：

- `gif_gifsicle_enabled`
- `gif_gifsicle_level`
- `gif_gifsicle_skip_below_kb`

#### B3. 渲染链路接入

顺序改成：

```text
ffmpeg render -> gifsicle optimize -> upload -> persist -> evaluation
```

#### B4. 指标与审计

将优化前后大小写入：

- `output.metadata.gif_optimization_v1`
- `evaluation.feature_json.gif_optimization_v1`
- `job.metrics.gif_optimization_summary_v1`

### 完成标准

- deliver GIF 平均体积下降
- rerender 路径也能跑优化
- 失败时自动回退到原始 GIF
- 审计链能看到优化结果

---

## 2.3 Epic C：PySceneDetect 旁路接入

### 目标

先让 PySceneDetect 产出 scene list，但不立刻强制替换现有 candidate 逻辑。

### 涉及文件

- `internal/videojobs/processor_scene_selection.go`
- `internal/videojobs/processor_gif_candidates.go`
- `internal/videojobs/processor_pipeline.go`
- `internal/videojobs/quality_settings.go`
- `internal/handlers/video_quality_settings.go`

### 子任务

#### C1. 新增工具封装

新增：

- `internal/videojobs/tool_scenedetect.go`
- `internal/videojobs/tool_scenedetect_test.go`

#### C2. 后台开关

新增设置：

- `scene_detect_enabled`
- `scene_detect_provider`
- `scene_detect_detector`
- `scene_detect_threshold`
- `scene_detect_min_scene_len_sec`

#### C3. 旁路 metrics

先不改变最终候选，仅记录：

- scene count
- avg scene duration
- fallback used
- scene boundaries

#### C4. 第二步切 candidate consumption

在确认旁路效果后，再让 `processor_gif_candidates.go` 消费 PySceneDetect 结果。

### 完成标准

- scene detect 失败时自动回退 ffmpeg scene logic
- audit chain 能看到 scene provider
- baseline 脚本能对比“旧候选 vs 新候选”

---

## 2.4 Epic D：VMAF 离线回归链路

### 目标

让 profile 变更、gifsicle 接入、候选策略变更，都可以通过离线基线回归比较。

### 涉及文件

- `internal/videojobs/tasks.go`
- `internal/videojobs/benchmark_vmaf.go`
- `internal/videojobs/tool_vmaf.go`
- `scripts/run_video_jobs_baseline_flow.sh`
- `scripts/compare_video_jobs_baseline.py`

### 子任务

#### D1. 新增 benchmark task

建议任务名：

- `video_jobs:benchmark_vmaf`

#### D2. 只跑 admin / baseline / offline

第一期不要接用户同步链路。

#### D3. 先写 JSON，后补表

第一期先落到：

- `job.metrics.vmaf_benchmark_summary_v1`
- `output.metadata.vmaf_benchmark_v1`

### 完成标准

- 基线脚本能输出 vmaf 对比结果
- rerender 后可以手动触发 benchmark
- 参数改动上线前能跑一轮回归

---

## 2.5 Epic E：平台可观测性最小集

### 目标

让第三方工具接入以后，定位失败、超时、体积异常、队列堆积都更容易。

### 涉及文件

- `cmd/worker/main.go`
- `internal/router/router.go`
- `internal/middleware/`
- `internal/videojobs/`
- `internal/service/`

### 子任务

#### E1. Asynqmon

上线一个内部运维可见的队列监控页。

#### E2. OpenTelemetry

至少给下面 4 类 span 打上：

- HTTP 请求
- Asynq task
- ffmpeg / ffprobe / gifsicle / PySceneDetect 外部命令
- AI 调用 / 七牛上传下载

### 完成标准

- 单任务链路能追到 tool 级别
- 队列积压、失败、重试有可视化入口
