# 配置、数据模型与迁移变更清单

> 目标：把第一期三方接入需要的配置、结构体、数据库变更一次性梳理清楚。

---

## 1. 配置层变更

当前配置集中在：`internal/config/config.go`

第一期建议新增以下字段：

## 1.1 Runtime / tool 配置

```text
GIFSICLE_BIN=
PYSCENEDETECT_PYTHON_BIN=
PYSCENEDETECT_BIN=
VMAF_ENABLED=
LIBVMAF_MODEL_PATH=
ENABLE_EXTERNAL_TOOL_FALLBACK=true
```

### 说明

- `GIFSICLE_BIN`：允许自定义二进制路径，不依赖 PATH
- `PYSCENEDETECT_PYTHON_BIN`：默认 `python3`
- `PYSCENEDETECT_BIN`：脚本入口，可为空，走 `python -m scenedetect`
- `VMAF_ENABLED`：是否允许离线 benchmark
- `LIBVMAF_MODEL_PATH`：模型路径
- `ENABLE_EXTERNAL_TOOL_FALLBACK`：外部工具失败是否回退本地逻辑

## 1.2 可观测性配置

```text
ASYNQMON_ENABLED=true
OTEL_ENABLED=true
OTEL_EXPORTER_OTLP_ENDPOINT=
OTEL_SERVICE_NAME=emoji-backend
```

## 7. 推荐字段约束

## 枚举字段

- `scene_detect_provider`: `ffmpeg | pyscenedetect`
- `scene_detect_detector`: `content | adaptive | threshold`

## 数值约束

- `gif_gifsicle_level`: `1..3`
- `gif_gifsicle_skip_below_kb`: `0..4096`
- `gif_gifsicle_min_gain_ratio`: `0..0.50`
- `scene_detect_threshold`: `1..100`
- `scene_detect_min_scene_len_sec`: `0.2..10`
- `vmaf_sample_rate`: `1..30`
- `vmaf_min_clip_sec`: `0.5..10`

---

## 8. 回滚友好设计

所有新增字段建议遵循：

- **有默认值**
- **默认关闭新能力，或默认保持旧行为**
- 工具缺失时能 fallback
- JSON 写入是增量字段，不改旧键含义

具体建议：

- `SceneDetectEnabled` 默认 `false`
- `VMAFBenchmarkEnabled` 默认 `false`
- `GIFGifsicleEnabled` 可默认 `true`，但失败必须 fallback 原产物
