# 06-新同事上手文档

> 适用对象：第一次接手 `emoji/backend` 的开发同学

## 1. 先知道你接手的是啥

这不是单纯“表情包管理后台”，而是一个综合后端：

- 内容库：合集 / 表情 / 分类 / 标签 / 下载
- 用户系统：短信登录、密码登录、风控
- 视频生产：视频转 GIF/WebP/JPG/PNG/live
- AI 子系统：GIF Director/Planner/Judge、Meme 生成
- 运营后台：健康看板、导出、质量配置、算力账户

一句话理解：

> **它是一个“内容平台 + 媒体生产引擎 + AI 运营后台”的 Go 单体系统。**

### 为什么这么读

## 3. 本地运行前的准备

## 3.1 依赖

你至少需要：

- Go 1.25.x
- PostgreSQL
- Redis
- 七牛配置
- `ffmpeg`
- `ffprobe`

如果你需要跑 meme 合成，还需要：

- OSS 配置
- 字体文件（默认 `assets/fonts/NotoSansSC-Bold.ttf`）

---

## 3.2 环境变量

先复制并修改：

- `.env.example` -> `.env`

重点配置：

- `DB_*`
- `REDIS_*`
- `ASYNQ_*`
- `QINIU_*`
- `JWT_*`
- `AI_*`
- `OSS_*`（可选）

> 注意：当前仓库里的 `.env.example` 需要你特别小心，安全审查文档里已经指出里面不应保留真实 key。

## 4. 本地启动顺序

## 4.1 启 API

```bash
go run ./cmd/api
```

默认端口：

- `APP_PORT=5050`

健康检查：

- `GET http://localhost:5050/healthz`

Swagger：

- `http://localhost:5050/swagger/index.html`

---

## 4.2 启 Worker

```bash
go run ./cmd/worker
```

Worker 负责：

- 消费视频任务
- 调 ffmpeg/ffprobe
- 上传产物
- 同步 public 表
- 结算积分与成本

没有 Worker，视频任务只能入队，不能执行。

---

## 4.3 跑测试

```bash
go test ./...
```

当前工作区测试是能通过的。

## 7. 新同事最容易迷糊的点

## 7.1 为什么同时有 `archive.*` 和 `public.*`

因为：

- `archive.*` 更像内部原始业务域
- `public.*` 更像前台产品查询域

特别是视频任务，public 层是为了给前台和反馈链路一个更稳定的模型。

---

## 7.2 为什么同一个视频任务会牵出这么多表

因为它不只是“转个 GIF”，还要记录：

- 原任务
- 产物
- 事件
- 候选窗口
- 评分
- AI 提案
- AI 复审
- Public 输出
- 用户反馈
- 成本
- 积分冻结/结算

这是系统最核心的复杂度来源。

---

## 7.3 为什么后台文件那么大

因为当前很多逻辑还没充分模块化，特别是：

- `internal/videojobs/processor.go`
- `internal/handlers/admin_video_jobs.go`

所以你日常开发要格外注意回归范围。

## 9. 本地调试建议

## 9.1 先看这几个日志点

- API 启动日志
- Worker 启动日志
- `video job started`
- `frame extraction completed`
- `animated render pipeline started`
- `video job completed`

## 9.2 出问题先区分是哪一层

### 如果创建任务就失败

先看：

- 路由校验
- source probe
- qiniu 配置
- ffmpeg/ffprobe 检查

### 如果任务一直 queued

先看：

- worker 是否启动
- Redis / Asynq 是否可用
- `media` 队列是否堆积

### 如果任务 running 后失败

先看：

- ffmpeg
- 源视频下载
- AI provider
- 七牛上传
- package zip
